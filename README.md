# web-grandlisboa-lab

## 项目简介

本仓库用于归档和发布多个独立的 HTML 页面。这些页面可能包含实验性的前端原型、文档展示、工具页面或静态内容集合。仓库不针对任何特定域名或具体网站，而是作为一个通用的 HTML 页面存储与展示空间。

## 目录说明

```
web-grandlisboa-lab/
├── pages/          # 存放各独立 HTML 页面文件
├── assets/         # 页面引用的静态资源（CSS、JS、图片等）
├── README.md       # 本文件
└── .gitignore      # Git 忽略规则
```

- `pages/`：每个子目录或文件对应一个独立的 HTML 页面，命名清晰以便识别内容用途。
- `assets/`：按需组织的资源文件夹，确保页面引用路径相对独立。

## 页面归档说明

- 每个 HTML 页面均为独立完整文件，可直接在浏览器中打开运行。
- 页面之间无强关联，互不依赖，便于单独维护和更新。
- 归档内容主要为静态页面，不涉及后端服务或数据库。
- 页面可能引用 CDN 资源，请确保网络连通性以正常加载外部库。

## 使用方式

1. 克隆仓库到本地：
   ```bash
   git clone https://github.com/your-username/web-grandlisboa-lab.git
   ```
2. 进入 `pages/` 目录，选择需要的 HTML 文件在浏览器中打开。
3. 若需部署到服务器，可将整个仓库或 `pages/` 目录作为静态站点根目录。

## 维护说明

- 欢迎提交 Pull Request 添加新的 HTML 页面，请确保文件结构清晰、资源引用正确。
- 提交前请检查页面是否包含绝对路径或敏感信息，建议使用相对路径引用资源。
- 仓库维护者会定期审查内容，移除无效或过时的页面。
- 如发现页面加载异常或资源缺失，可提交 Issue 反馈。
